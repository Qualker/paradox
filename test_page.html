Chrome LNA PoC

Sanday × Vitalograph — Local Network Access

Validação de comunicação directa entre site HTTPS e software local HTTP,
sem device-server, usando a especificação Chrome LNA (v142+).

⚠️ Chrome LNA requer Chrome 142 ou superior. A tua versão pode não suportar este fluxo.

Como funciona: O Chrome intercepta o fetch para localhost (rede privada) e primeiro envia um preflight OPTIONS com o header Access-Control-Request-Private-Network: true. O servidor local tem de responder com Access-Control-Allow-Private-Network: true. Só depois o request real é enviado.

📝 O que este PoC valida
✅ Sucesso significa: O Chrome 142+ enviou o preflight LNA automaticamente, o servidor local respondeu com Access-Control-Allow-Private-Network: true, e o fetch de HTTPS para HTTP localhost funcionou sem device-server.
⚠️ Limitações conhecidas: LNA é Chrome-only (Safari não suporta). O Chrome pode mostrar um prompt de permissão ao utilizador a partir de versões futuras. Testar neste ficheiro aberto localmente (file://) simula o comportamento — para o teste real, serve esta página via HTTPS.
